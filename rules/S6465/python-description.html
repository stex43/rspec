Why is this an issue?

Unpacking lets developers assign values of an iterable simultaneously to different targets in a single assignment. However, for this assignment to work, the iterable should have the same number of elements as there are targets in the target list. If this is not respected, a ValueError will be produced at runtime.

Noncompliant code example

def foo(param):
  ls = [1, 2, 3]
  x, y = ls # Noncompliant

Compliant solution

def foo(param):
  ls = [1, 2, 3]
  x, y, z = ls
